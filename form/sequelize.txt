Sequelize - It is a ORM (Object relational Mapping) library for node.js used to 
            interact with database using js methods and functions instead of sql code.

Model - Database table structure
=> const user=sequelize.define(user,{
    name:DataTypes.STRING,
    age:DataTypes.INTEGER
});
Datatypes - string,text,integer,float,double,boolean,date,dateonly,time,char,json

CRUD operations
=>Create
    await User.create{(
        name:"Pari",
        age:21
    )}

=>Read
    await users=User.findAll()
    console.log(users)

=>update
    await User.update(
        {age:21},
        {where:{name:"pari"}}
    )

=>Delete
    await User.destry({
        where:{name:"Pari"}
    })

Validation - checks the data before saving to db
    email:{
        type:DataTypes.STRING,
        allowNull:false
    }

Async and await support - waits until the database operation get completed
const users=await findAll()

Important things in sequelize

workflow:
Postman creates req => express server receives res => check routes => controller function call => 
                        function execute => sequelize create query => data saved in postgresql => response

Query filters in sequelize

1.where
User.findAll({
    where:{age:21}
});

2.where (multiple conditions)
User.findAll({
    where:{age:21,
    name:"Pari"
    }
});

3.Operators
const {op}=require("sequelize");
=>Greater than
User.findAll({
    where:{age:[op.gt]:18}
});

=>Less than
User.findAll({
    where:{age:[op.lt]:18}
});

=>IN
User.findAll({
    where:{age:{[op.in]:[18,19,20]}
    }
});

=>like
User.findAll({
    where:{name:{[op.like]:"%p"}}
});

4.LIMIT
User.findAll({
    limit:5
});

5.OFFSET
User.findAll({
    limit:5,
    offset:4
});

6.order
User.findAll({
    order:[["age","ASC"]]
});

7.select specific attriibutes
User.findAll({
    attributes:["name","age"]
});

API - Application programming Interface
API acts like a bridge b/w frontend and backend

Authentication in API
1.Session based Authentication
    =>User logs in
    =>Server creates the Session
    =>server stores the user info
    =>browser gets the session id (cookie)
    =>Every req uses the session id

2.JWT - secure token used to identify a user after login
    If user details are correct a token is generated and token is sent to client ,
    every req uses token if valid then allowed.

3.API Key - secret key used to access the API
    Developer gets the secret api key
    It is sent with every request (req+key)
    If valid -> returns data,else reject->req

4.OAuth - authorization using third party application to access user data
